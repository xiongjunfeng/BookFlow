# 图书管理系统MVP开发计划

## 1. 项目概述

基于需求分析文档和技术选型方案，我们将开发一个前后端分离的图书管理系统MVP版本，采用Vue 3 + Flask + SQLite技术栈，确保在1小时内完成核心功能。

## 2. 技术栈

| 模块 | 技术选型 |
|------|----------|
| 前端 | Vue 3 + Vite + Element Plus + Axios |
| 后端 | Python + Flask + SQLAlchemy + JWT |
| 数据库 | SQLite |

## 3. 核心功能（简化版）

为确保在1小时内完成，我们将实现以下核心功能：

### 3.1 后端功能
- 用户认证（登录/注册）
- 图书CRUD操作
- 图书借阅/归还
- 借阅历史查询

### 3.2 前端功能
- 登录页面
- 图书列表页面
- 借阅记录页面
- 基本的用户界面

## 5. 详细任务分解

### 5.1 项目结构搭建（5分钟）
- 创建backend目录及子目录
- 创建frontend目录及子目录
- 创建database目录

### 5.2 后端项目初始化（8分钟）
- 创建requirements.txt
- 安装Flask、SQLAlchemy、JWT等依赖
- 配置Flask应用

### 5.3 数据库模型设计（7分钟）
- 用户表（id, username, password, role）
- 图书表（id, book_no, title, author, stock）
- 借阅记录表（id, user_id, book_id, borrow_date, return_date, status）

### 5.4 后端API实现（15分钟）
- 认证API：注册、登录
- 图书API：获取列表、借阅、归还
- 借阅记录API：获取用户借阅记录

### 5.5 前端项目初始化（8分钟）
- 创建Vue 3项目
- 安装Element Plus、Axios等依赖
- 配置Vue Router

### 5.6 前端页面实现（12分钟）
- 登录页面
- 图书列表页面（包含借阅/归还按钮）
- 借阅记录页面

### 5.7 测试与调试（5分钟）
- 测试用户注册登录
- 测试图书借阅归还
- 测试数据一致性

## 6. 需求简化说明

为确保在1小时内完成，我们对原需求进行了以下简化：

1. **用户管理**：仅保留基本注册登录功能，简化角色管理
2. **图书管理**：仅保留核心字段（编号、书名、作者、库存），简化其他字段
3. **UI设计**：使用Element Plus基本组件，简化样式
4. **搜索功能**：仅实现基本的图书列表展示，暂不实现搜索
5. **权限控制**：简化权限检查，仅区分管理员和普通用户

## 7. 预期成果

1. 可运行的前后端分离项目
2. 基本的用户认证功能
3. 图书借阅归还功能
4. 借阅历史记录查询
5. 数据一致性保证

## 8. 风险与应对

| 风险 | 应对措施 |
|------|----------|
| 依赖安装问题 | 提前准备好requirements.txt和package.json |
| API开发超时 | 优先实现核心功能，简化非必要API |
| 前端页面复杂 | 使用Element Plus组件库，快速搭建页面 |
| 数据一致性问题 | 实现事务管理，确保借阅归还与库存同步 |
